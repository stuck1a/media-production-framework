# Project Agent Onboarding

Before doing project work, read `docs/developer-guide.md`.

Then follow the document order defined there. The current onboarding sequence is:

1. `docs/developer-guide.md`
2. `docs/implementation-plan.md`
3. Relevant ADRs in `docs/adr/`
4. `docs/coding-standards.md`
5. `docs/project-structure.md`
6. `docs/testing.md`
7. `CONTRIBUTING.md`
8. `docs/capabilities.md`
9. `docs/release-process.md`
10. `docs/roadmap.md`

If `docs/developer-guide.md` changes, treat that file as authoritative and follow its updated order.

Use `docs/roadmap.md` to identify the current milestone before continuing implementation work.
